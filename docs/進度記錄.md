> 最後更新：2026-07-11

## 專案目標

在正式進入 Agent 決策層之前，先驗證三種不同架構的 2D 分割模型能各自在獨立環境跑通「輸入胸部 X 光 → 輸出氣胸 mask」的完整路徑。資料集為 **SIIM-ACR Pneumothorax Segmentation**。三模型：

| 模型 | 架構路線 | 取得方式 |
|---|---|---|
| **nnU-Net (2D)** | CNN 自動配置 | 在 SIIM-ACR 上從頭訓練 |
| **Swin-UNet** | Transformer | 官方 repo + ImageNet 預訓練 encoder，從頭訓練 |
| **MedSAM** | prompt-based foundation model | 官方 pretrained checkpoint，用 nnU-Net 預測框當 box prompt |

驗證目標是 pipeline 跑通 + 訓練確實學到東西（非追求 SOTA）。

## 環境資訊

- **機器**：RTX 5080（16GB VRAM，Blackwell / compute capability 12.0，driver 591.86 / CUDA 13.1）、WSL2、系統 Python 3.14（無 conda）

  - `swinunet-env` — Swin-UNet 相依（timm 1.0.27 等）
  - `medsam-env` — segment-anything + skimage

## 結果（test 集，532 張，其中陽性 266 張）

| 模型 | Mean Dice（全部） | Mean Dice（陽性） | 訓練量 |
|---|---|---|---|
| **nnU-Net (2D)** | 0.6315 | **0.5187** | 1000 epochs（~8.7h） |
| **MedSAM**（借 nnU-Net 框） | **0.6324** | 0.5053 | zero-shot（官方 checkpoint） |
| **Swin-UNet** | 0.5995 | 0.3607 | 150 epochs（~45min） |

**解讀**：

- **nnU-Net 最穩**：陽性 Dice 0.52。氣胸病灶細長稀疏，單 fold 這個數字屬合理水準（內部驗證 fold 的 Mean Validation Dice 為 0.4249，子集與空樣本計法不同故略低）。當初選它的框餵 MedSAM 的判斷是對的。
- **MedSAM 傾向過度分割**：陽性 Dice 0.51，略低於 nnU-Net。它繼承 nnU-Net 的框定位、只精修邊界，但**會填滿整個框**——疊圖可見藍色沿整個側胸壁大範圍擴張。框越鬆，過度分割越明顯。
- **Swin-UNet 最弱**：陽性 Dice 僅 0.36。疊圖可見部分案例完全沒抓到（Dice = 0.000，只吐零散碎塊）。原因符合預期：transformer 需要更多資料，且 224×224 的輸入解析度對細長氣胸線條的細節保留不利（nnU-Net / MedSAM 都在 1024×1024 作業）。

**疊圖驗證**：`outputs/verification/test_overlays.png`（四欄：GT 綠 / nnU-Net 紅 / MedSAM 藍 / Swin-UNet 黃，標註各自 Dice）

三個模型都能穩定產出解剖位置正確的 mask，**pipeline 端到端驗證完成**。

## 關鍵問題與解法（踩坑記錄）

1. **RTX 5080 太新（sm_120）**：最大技術風險。用 PyTorch cu128 build，每個環境裝完都實跑一次 GPU 矩陣乘法確認 kernel 真能執行（非只看 `is_available()`）。三環境全通過。
2. **Kaggle 新版 token**：`KGAT_` 開頭的單一 token，舊版 `kaggle` CLI 只認 kaggle.json，改用 `kagglehub` + `~/.kaggle/access_token`。`.bashrc` 的 export 因非互動式 shell 讀不到，故寫入檔案。
3. **競賽檔缺訓練影像**：官方 `siim-acr-pneumothorax-segmentation` 下載只有未標註測試影像 + RLE 標註 CSV，訓練 DICOM 當年須從已停用的 GCP Healthcare API 抓。改用 Kaggle 鏡像 `jesperdramsch/siim-acr-pneumothorax-segmentation-data`（原始 DICOM，SOP UID 對得上官方 `stage_2_train.csv` 標註）。
4. **下載中斷損壞**：第一次下載被截斷（289MB/412MB，ZIP 損壞），確認後刪除重下。
5. **RLE 格式**：官方 `mask_functions.py` 用「相對偏移」而非絕對 index，初版寫錯，修正後對照官方解碼器 50/50 完全一致、round-trip 精確。
6. **torchvision 版本衝突**：nnunetv2 安裝時從 PyPI 拉到非 cu128 的 torchvision，導致 `torchvision::nms` 算子不存在。強制重裝 cu128 版解決。
7. **Swin npz 路徑**：官方 `train.py` 只在 `--dataset Synapse` 時才把 `train_npz` 附加到 root_path，我們用 `pheumo` 故 npz 需直接放 `--root_path` 下。

## 各模型執行指令備忘

```bash
# nnU-Net（nnunet-env）
source nnunet/env.sh
nnUNetv2_train 1 2d 0                        # 訓練（已完成）
nnUNetv2_predict -i .../imagesTs -o outputs/nnunet_preds -d 1 -c 2d -f 0 --disable_tta
python common/mask_to_bbox.py --masks-dir outputs/nnunet_preds --out-json outputs/nnunet_preds/bboxes.json

# Swin-UNet（swinunet-env）
python swin_unet/make_npz.py --img-size 224 --dataset-name pneumo
cd swin_unet/repo && python train.py --dataset pneumo --n_class 2 --img_size 224 \
  --root_path ../npz_data --output_dir ../checkpoints \
  --cfg configs/swin_tiny_patch4_window7_224_lite.yaml --max_epochs 150 --batch_size 24
python swin_unet/infer_swin.py --checkpoint swin_unet/checkpoints/best_model.pth

# MedSAM（medsam-env）
python medsam/infer_with_box.py --bboxes outputs/nnunet_preds/bboxes.json \
  --checkpoint medsam/repo/work_dir/MedSAM/medsam_vit_b.pth

# 評估（任一 env）
python common/evaluate.py --preds-dir outputs/<model>_preds --name <model>
```
